## Detailed Findings

### P0 — Blocking

**[P0] Side-stripe card border — absolute banned pattern**
- **Location**: `components/RestaurantCard.tsx` line 21–22
- **Category**: Anti-Pattern
- **Impact**: This is the top-listed banned CSS pattern in the impeccable skill. Hard-coded `w-2 bg-[#E46C0A]` absolute-positioned stripe is the most recognizable AI design tell across admin/food/dashboard UIs.
- **Recommendation**: Remove the stripe entirely. Signal "visited" status with a background tint on the whole card, or a small badge/icon in the corner. Do NOT replace with `box-shadow inset`.
- **Suggested command**: `/colorize` or `/polish`

**[P0] Inter font — on reflex_fonts_to_reject list**
- **Location**: `app/layout.tsx` line 2
- **Category**: Anti-Pattern
- **Impact**: Inter is the canonical AI-slop default font. Directly contradicts "handmade, healing, local" brand personality.
- **Recommendation**: Replace with a font that matches the warm/handmade brief. The font selection procedure must be run before picking a replacement.
- **Suggested command**: `/typeset`

### P1 — Major

**[P1] Hard-coded colors throughout — token system bypassed**
- **Location**: `RestaurantCard.tsx` (`bg-[#E46C0A]`, `bg-[#E0E0E0]`, `text-blue-500`, `text-yellow-500`, `bg-green-100`, `text-green-700`), `restaurants/page.tsx` (`bg-[#E46C0A]`, `#F5F5F5`, `#4A4A4A`, `#666666`), `page.tsx` (inline `style={{ backgroundColor: '#E46C0A' }}`)
- **Category**: Theming
- **Impact**: The orange accent can't be changed in one place; dark mode (if ever added) will break; brand evolution requires hunting across 5+ files.
- **Recommendation**: Define `--color-brand-orange`, `--color-visited`, `--color-proximity`, `--color-rating` as CSS custom properties in `globals.css`, then wire to Tailwind tokens. Replace all hard-coded values.
- **Suggested command**: `/colorize`

**[P1] Zero-chroma token palette — no warmth in the system**
- **Location**: `app/globals.css` lines 54–76
- **Category**: Theming
- **Impact**: Every OKLCH token has `0` chroma — pure neutral achromatic. The design brief explicitly says "tint everything toward amber." The warm feeling is entirely absent from the token layer.
- **Recommendation**: Add a small amber chroma (`0.008–0.015`, hue `~60-75`) to all neutral tokens — background, card, muted, border — to create subconscious warmth cohesion with the orange accent.
- **Suggested command**: `/colorize`

**[P1] Radio inputs and range slider not labelled accessibly**
- **Location**: `recommend/page.tsx` lines 108–116 (radio group has no `<fieldset>`/`<legend>`), `RestaurantForm.tsx` line 103 (range input has no `aria-label` or `aria-valuetext`)
- **Category**: Accessibility
- **Impact**: Screen reader users can't understand the radio group purpose or the current proximity value. WCAG 2.1 AA violation (1.3.1, 4.1.2).
- **Recommendation**: Wrap radio group in `<fieldset><legend>已造訪篩選</legend>`. Add `aria-label` and `aria-valuetext={PROXIMITY_LABELS[proximity]}` to the range input.
- **Suggested command**: `/harden`

**[P1] Interactive `<button>` elements without visible focus indicators**
- **Location**: Tag toggle buttons in `recommend/page.tsx` line 128, filter tab buttons in `restaurants/page.tsx` line 61, edit/delete buttons in `RestaurantCard.tsx` lines 87–98
- **Category**: Accessibility
- **Impact**: Keyboard-only users can't see which element has focus. WCAG 2.1 AA violation (2.4.7).
- **Recommendation**: Add `focus-visible:ring-2 focus-visible:ring-offset-2 focus-visible:ring-[--color-brand-orange]` to all interactive elements.
- **Suggested command**: `/harden`

**[P1] `window.confirm()` used for destructive action**
- **Location**: `app/restaurants/page.tsx` line 22
- **Category**: Accessibility / Anti-Pattern
- **Impact**: `confirm()` blocks the main thread, can't be styled, is inaccessible to screen readers in some contexts, and looks completely at odds with any visual design.
- **Recommendation**: Replace with an inline confirmation state on the card (clicking Delete shows "確定刪除？ [是] [否]" in place).
- **Suggested command**: `/harden`

### P2 — Minor

**[P2] Touch targets < 44×44px throughout mobile UI**
- **Location**: Edit/Delete buttons (`RestaurantCard.tsx` lines 88, 95: `text-xs`), Rating buttons (`RestaurantForm.tsx` line 121: `w-10 h-10` = 40px), Tag remove `X` button (`RestaurantForm.tsx` line 148)
- **Category**: Responsive
- **Impact**: Hard to tap accurately on phones — the primary device for this app. Apple HIG and WCAG 2.5.5 recommend ≥44×44px.
- **Recommendation**: Increase edit/delete to at least `p-2` with larger text. Rating buttons to `w-11 h-11`. Tag remove needs larger hit area via padding.
- **Suggested command**: `/adapt`

**[P2] `<input type="range">` unstyled — broken on iOS Safari**
- **Location**: `RestaurantForm.tsx` line 103, `recommend/page.tsx` (proximity slider)
- **Category**: Responsive
- **Impact**: iOS Safari renders range sliders with system chrome that ignores custom styling.
- **Recommendation**: Add CSS for `-webkit-appearance: none` with custom track/thumb styles, or replace with a custom slider component.
- **Suggested command**: `/adapt`

**[P2] No loading skeleton on restaurant list**
- **Location**: `app/restaurants/page.tsx` line 35: `<p>載入中...</p>`
- **Category**: Performance / UX
- **Impact**: Plain text loading state feels unfinished; layout jumps on slower connections.
- **Recommendation**: Replace with skeleton cards matching the RestaurantCard structure.
- **Suggested command**: `/delight`

**[P2] `Label` component rendered without `htmlFor` on some inputs**
- **Location**: `RestaurantForm.tsx` lines 97, 101
- **Category**: Accessibility
- **Impact**: Clicking the label won't focus the input; screen readers may not associate them.
- **Recommendation**: Add explicit `htmlFor` matching input `id` attributes on all form fields.
- **Suggested command**: `/harden`

**[P2] `<p>` used as section header above interactive `<details>` elements**
- **Location**: `app/page.tsx` line 23
- **Category**: Accessibility
- **Impact**: Semantic mismatch; the "更新記錄" header should be a proper `<h2>`.
- **Recommendation**: Replace `<p>` with `<h2>`.
- **Suggested command**: `/polish`

**[P2] Inconsistent button hierarchy — every action looks equally important**
- **Location**: Multiple pages — edit, delete, choose all share equal visual weight
- **Category**: Anti-Pattern
- **Impact**: Users can't instantly identify the primary action.
- **Recommendation**: "就選這家！" = primary. Edit = ghost/text. Delete = text in muted color.
- **Suggested command**: `/distill`

**[P3] Flat type hierarchy across all pages**

- **Suggested command**: `/typeset`
